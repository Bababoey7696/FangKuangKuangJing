# 方块矿境长期开发指南

## 一、每次开发的固定流程

1. 从 `develop` 新建分支，例如 `feature/new-skill` 或 `fix/menu-layout`。
2. 一次分支只解决一类问题，避免速度、界面、存档同时大改。
3. 修改后至少执行 `./gradlew assembleDebug`。
4. 覆盖安装测试版，确认旧存档仍存在。
5. 提交 Pull Request 合并到 `develop`。
6. 准备发版时，把 `develop` 合并到 `main`，提升 `versionCode` 和 `versionName`。
7. 使用原长期 JKS 在本地签名 APK，并把 APK 作为 GitHub Release 附件发布。

## 二、版本规则

- 小修复：1.4.1、1.4.2；每次仍必须增加 `versionCode`。
- 普通功能版本：1.5、1.6。
- 大规模架构或玩法变化：2.0。
- 不得重复使用已经发布过的 `versionCode`。

## 三、代码结构方向

- `BlockTetrisView`：只负责绘制、输入和游戏状态协调。
- `GameBalance`：速度、落地缓冲、技能消耗和冷却。
- `RunMissions`：单局任务与奖励。
- `ThemeCatalog`：皮肤名称、配色和解锁条件。
- 下一阶段建议继续拆分 `BoardModel`、`PieceController`、`SaveRepository` 和 `GameRenderer`。

## 四、测试重点

- 覆盖安装后 SharedPreferences 数据是否保留。
- 标准模式 20、30、40 级触控是否仍可操作。
- 暂停、切后台、冻结结束后是否瞬间锁死。
- 全面屏、小屏幕下按钮和文字是否越界。
- 任务奖励是否重复发放，续分开局是否错误完成任务。

## 五、签名密钥

JKS 不进入 GitHub，也不发到公开聊天。至少保留两份离线备份，分别存放；同时离线记录别名、证书指纹和密码。
